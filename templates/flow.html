夢日記生成システム - 動き時系列

1. ユーザー入力

夢の種（例: "火星の植樹都市で眠る旅行者"）を入力

2. 外部トリガー

HTTP POST → repository_dispatch (event_type: generate-dream)

3. GitHub Actions起動

workflow: generate-dream.yml が実行される

4. Sakura APIリクエスト

POST https://api.ai.sakura.ad.jp/v1/chat/completions
種を含むプロンプトを送信、星新一風SF短編を生成

5. レスポンス受信

400字に正規化 → JSONとMarkdownを生成

6. Wiki更新

GitHub Wikiをクローン → JSON/MDファイルを追加 → コミット&プッシュ

7. Supabase保存（オプション）

PostgreSQL (Supabase) の dreams テーブルに保存
Flaskサーバーからも読み書き可能

8. 閲覧

Wikiページまたは /api/dreams で夢日記を閲覧

サーバーAPIエンドポイント

  • GET /api/dreams - 夢日記一覧
  • GET /api/dream/random - ランダム生成・保存
  • POST /api/seed/submit - 種を投稿
  • GET /api/seeds - 種一覧
